# Shell Script 실전 Tip

## 1. echo에서 역슬래시(\)의 의미

```bash
echo "예시 : $0 result.txt \"내용을 입력하세요!\""
여기서 "처럼 **역슬래시()**를 사용하는 이유는

따옴표(쌍따옴표) 자체를 문자열 안에 문자로 넣고 싶을 때

"내용을 입력하세요!"를 실제 문자열 그대로 보여주고 싶을 때

즉, bash는 문자열을 쌍따옴표(")로 감싸기 때문에,
문자열 안에 또 " "가 오면 구분을 못함
그래서 이스케이프 문자(역슬래시)를 붙여 "로 표기
→ bash가 이걸 “진짜 문자로서의 쌍따옴표”로 인식


echo "hello \"world\""
# 출력: hello "world"
```
## 2. if문마다 exit 1, exit 2, exit 3 쓰는 이유
```bash
**exit [번호]**는 프로그램(스크립트) 종료 + 종료 상태코드 반환

일반적으로 0: 성공, 0이 아닌 숫자: 에러
→ 운영체제나 다른 프로그램/자동화툴이 종료코드로 성공/실패를 식별

왜 각기 다른 번호(1, 2, 3)로?

1: 일반적 인자 에러, 사용법 오류

2: 입력값(파일명/내용) 자체가 비었을 때

3: 파일 생성 실패(시스템 문제, 권한 등)

즉,
각 종료상태마다 “이유”를 명확하게 구분
자동화 환경, 배치, 파이프라인에서 스크립트의 실패 원인 분석에 도움

예시
리눅스/유닉스 관습적으로

exit 1: 인자 문제

exit 2: 환경/값 문제

exit 3: 실행/동작 문제

상태코드는 자유롭게 정의 가능!
(단, 0=성공, 1=실패라는 최소 규칙만 많이 지킴)
```

## 3. Shell에서 많이 쓰는 연산자(비교, 논리 등)
```bash
1) 산술 연산자
: 더하기

: 빼기

: 곱하기

/ : 나누기

% : 나머지

2) 비교 연산자
-eq : 같음 (equal)

-ne : 같지 않음 (not equal)

-gt : 큼 (greater than)

-lt : 작음 (less than)

-ge : 크거나 같음 (greater or equal)

-le : 작거나 같음 (less or equal)


[ "$a" -eq "$b" ]   # a == b
[ "$a" -gt 5 ]      # a > 5
3) 문자열 비교 연산자
= : 같음

!= : 다름

-z : 길이가 0인가

-n : 길이가 0이 아닌가


[ "$a" = "$b" ]    # 문자열 같음
[ "$a" != "$b" ]   # 문자열 다름
[ -z "$a" ]        # 빈 문자열인가
[ -n "$a" ]        # 빈 문자열이 아닌가
4) 파일/디렉터리 테스트 연산자
-f 파일 : 일반파일인가

-d 디렉터리 : 디렉터리인가

-e 파일 : 존재하는가

-r 파일 : 읽기 가능한가

-w 파일 : 쓰기 가능한가

-x 파일 : 실행 가능한가


[ -f myfile.txt ]  # myfile.txt가 일반 파일인가
[ -d /home/user ]  # /home/user가 디렉터리인가
5) 논리 연산자
-a : 그리고(and)

-o : 또는(or)

! : not(부정)


[ -f myfile.txt -a -r myfile.txt ] # myfile.txt가 존재하고, 읽기 가능한가
[ -z "$a" -o -z "$b" ]            # a 또는 b가 빈 문자열인가
6) 괄호 (복합 조건식)

if [ "$a" = "foo" ] && [ "$b" = "bar" ]; then
  # 두 조건 모두 참
fi
혹은 [[ ... ]]에서는 &&, || 사용 가능.

```
